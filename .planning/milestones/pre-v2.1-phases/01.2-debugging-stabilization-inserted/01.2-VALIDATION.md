# Phase 01.2: Debugging & Stabilization - Validation Strategy

This document outlines the testing protocols for the stabilization fixes implemented in Phase 01.2.

## Automated Tests

### Unit & Integration
- **Notification State**: Verify `hasUnreadSignals` is false when `isNervalisOpen` is true.
- **Vote Locking**: Mock `submitVote` with a delay and verify subsequent calls are ignored.

### UI / Browser Tests
- **Boundary Check**: Simulate drag events to coordinates outside the viewport; verify terminal position remains within bounds.
- **Z-Index Audit**: Use browser `getComputedStyle` to verify `z-index` of NervalisOverlay >= 100.

## Manual Verification (UAT)

### 1. Terminal Persistence & Boundaries
- [ ] Drag terminal to the top-left corner. It should stop at (0,0).
- [ ] Drag terminal to the bottom-right. It should stop exactly at the screen edge.
- [ ] Close and reopen terminal. It should persist its position.

### 2. Notification Sync
- [ ] Trigger a "New Signal" (e.g., via a poll or message).
- [ ] Observe the HUD notification indicator.
- [ ] Open the terminal.
- [ ] **Acceptance Criteria**: The HUD indicator disappears immediately upon opening.

### 3. Layering
- [ ] Open the terminal.
- [ ] Trigger a Privacy Poll or Stealth Entry dialog.
- [ ] **Acceptance Criteria**: The terminal remains on top of the dialog. You should be able to drag the terminal over the dialog.

### 4. World-State Synchronization
- [ ] Locate an evidence item in the 3D room.
- [ ] Open terminal, go to EVIDENCE tab.
- [ ] Resolve the poll as "ACCEPTED".
- [ ] **Acceptance Criteria**: The evidence sprite disappears from the 3D room view immediately.

### 5. Double-Voting Prevention
- [ ] Find an active poll.
- [ ] Rapidly double-click the "SÍ" or "NO" button.
- [ ] **Acceptance Criteria**: Only one vote is registered; the button becomes disabled/dimmed after the first click.
