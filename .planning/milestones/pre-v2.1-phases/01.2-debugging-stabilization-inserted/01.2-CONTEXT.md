<decisions>
## Implementation Decisions

### 1. Notification Synchronization
- **Trigger**: The "New Signal" HUD notification must clear immediately when the Nervalis terminal is opened, regardless of which tab is active.
- **State**: Ensure `hasUnreadSignals` is reset to `false` in `useTmaStore` on terminal toggle.

### 2. Layering & Interaction
- **Z-Index Strategy**: The Nervalis terminal (`NervalisOverlay`) MUST take absolute priority over all other UI elements (Modals, Popups, Overlays).
- **Drag Constraints**: The draggable terminal window must be restricted to the viewport boundaries. It cannot be dragged partially off-screen.

### 3. World-State Sync (QA Improvement)
- **Evidence Visibility**: Once an evidence poll is `ACCEPTED`, the corresponding `EvidenceSprite3D` must be hidden in the 3D room view.
- **Mechanism**: Update `resolvePoll` to set `is_hidden = true` on the evidence record when accepted, and ensure `InsideRoomArena` filters its `clues` state accordingly.

### 4. Logic Hardening
- **Double-Voting Bug**: Prevent users from submitting multiple votes on the same poll by adding a loading/locked state (`isVoting`) during the `submitVote` async call in `EvidenceTab.tsx`.

### 5. Performance
- **Aesthetic Stability**: Continue using CSS-only CRT scanlines and Framer Motion for glitch effects. No "Performance Mode" toggle needed as no lag is currently reported.

</decisions>
