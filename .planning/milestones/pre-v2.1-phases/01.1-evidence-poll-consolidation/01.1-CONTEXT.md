# Phase 01.1: Evidence Poll Consolidation - Context

**Gathered:** 2026-04-13
**Status:** Ready for planning

<domain>
## Phase Boundary
Migrate the evidence polling logic currently housed in the legacy modal `GlobalPollOverlay.tsx` into the unified `NervalisOverlay` terminal to resolve UI conflicts and system notification failures.
</domain>

<decisions>
## Implementation Decisions

### 1. Tab Structure & Purpose
- Rename the existing `POLLS` tab to `EVIDENCE`.
- This new tab will have **two functions**: 
  1. Handle active evidence voting protocols.
  2. Act as a searchable/viewable repository of all accepted evidence for students to browse.

### 2. Visual Theme Hand-off
- **Hybrid Alert Styling**: When an evidence poll is pending/active, the tab's internal UI should switch to a high-alert Red theme to maintain tension.
- When viewing the repository of accepted evidence, use the clean, standard Nervalis Blue.

### 3. Terminal Auto-Focus & Notifications
- **No forced tab switching**. When an evidence poll triggers, simply blink the `EVIDENCE` tab icon to alert the user.
- This will natively trigger the global hud notification (if the terminal is closed) and eliminate jarring interruptions.

### 4. Waiting State Feedback
- After a user votes, the tab should display a suspenseful terminal readout showing the current consensus status (who has voted vs. waiting), keeping immersive tension high until resolved.

### 5. Outbound Migrations (Other Poll Types)
- **Privacy Polls**: Remove them from the Nervalis `POLLS` tab. Return them to being a standard screen popup/modal, as they functionally block entry to a room.
- **Assassin Polls**: Move the assassin/volunteer logic entirely into the `ADMIN` tab. They do not belong in the public Evidence tab.

</decisions>
