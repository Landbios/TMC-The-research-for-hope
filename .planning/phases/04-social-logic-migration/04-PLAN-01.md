# Plan: Phase 04 - Multi-Channel Chat Integration

Refine the social interaction layer by integrating room-specific chat into the Nervalis terminal.

## Wave 1: Dynamic Chat Channels

### [1] Upgrade VerticalChatLog to Multi-Channel
Modify `VerticalChatLog.tsx` to support both GLOBAL and LOCAL_ROOM channels.
<read_first>
- `apps/tma-client/src/features/dashboard/components/VerticalChatLog.tsx`
- `apps/tma-client/src/store/useTmaStore.ts`
</read_first>
<action>
1. Add a `channel` state ('GLOBAL' | 'LOCAL') to `VerticalChatLog`.
2. Map `currentRoomId` dynamically:
   - If 'GLOBAL': Use `00000000-0000-0000-0000-000000000000`.
   - If 'LOCAL': Use `useTmaStore.getState().currentRoomId`.
3. Add a channel selector UI (two small buttons) at the top of the chat area.
4. Update the subscription logic to re-subscribe when the channel changes or when `currentRoomId` in the store changes.
5. Apply different styles to messages:
   - GLOBAL: Blue-tinted borders.
   - LOCAL: Green-tinted borders.
</action>
<acceptance_criteria>
- Terminal chat shows a "GLOBAL" and "LOCAL" toggle.
- Switching to LOCAL shows only messages from the player's current room.
- Changing rooms while in LOCAL mode automatically updates the chat to the new room.
</acceptance_criteria>

## Verification Plan
